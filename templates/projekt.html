CareBridge
Ein Praxisprojekt von Aaron Quazi
Examinierter Pflegefachmann Außerklinische Intensivpflege Beatmungspflege Healthcare IT

Dieses Projekt entstand aus dem Alltag der Beatmungspflege. Es zeigt, wie papierbasierte Pflegeprozesse digitalisiert werden können — praxisnah, prozessorientiert und mit echtem Fachverständnis.

Demo starten → Das Problem ↓ Über den Autor ↓
Das Ausgangsproblem
Beatmungspflege heute: Viele Formulare, viele Lücken, kein System das prüft
📄 So läuft es heute (papierbasiert)
📄 Intensivkurve (Papier)
→ Vitalwerte von Hand eingetragen
→ Kein automatischer Alarm bei Lücken
📄 Leistungsnachweis (Papier)
→ Handzeichen der Pflegekraft
→ Keine Zeitstempel-Kontrolle
📄 Beatmungsverordnung (Papier)
→ Alarmgrenzen nicht digital verfügbar
→ Keine automatische Prüfung
📄 Medikamentenplan (Papier)
→ Getrennt vom restlichen Dokumentationssystem
→ Keine Wechselwirkungs-Hinweise
❌ Problem: Mehrfachdokumentation, Lücken werden erst bei der Übergabe erkannt, kein System prüft Vollständigkeit.
✅ So funktioniert CareBridge (digital)
✅ Intensivkurve digital
→ Pflichtfelder rot wenn überfällig
→ Automatische 2h-Intervall-Kontrolle
✅ Leistungsnachweis digital
→ Kürzel + Zeitstempel bei jedem Eintrag
→ Zeitfenster-Prüfung (zu früh / pünktlich / zu spät)
✅ Beatmungsverordnung digital
→ Alarmgrenzen aus ärztlicher Verordnung
→ SpO₂-Warnung direkt in der Kurve
✅ Compliance Engine
→ Dashboard zeigt offene Dokumentationen
→ Übergabe-Ampel bei kritischen Lücken
✅ Lösung: Ein System prüft Vollständigkeit in Echtzeit — Fehler werden verhindert, nicht erst bei der Übergabe entdeckt.
Was CareBridge kann
12 Module — alle entwickelt aus realer Pflegepraxis
📊
Dashboard
Patientenübersicht, Beatmungsstatus, Compliance-Ampel, offene Aufgaben auf einen Blick
📈
Intensivkurve
Editierbare Stundenansicht 07–22 Uhr, Pflichtfeld-Rot bei Überfälligkeit, SpO₂-Alarmgrenzen visuell
✅
Leistungsnachweis
Pflegeleistungen abhaken mit Kürzel, Zeitfenster-Kontrolle, Begründung bei Verspätung, Rückgängig-Funktion
🔄
Übergabe
SBAR-Format mit echten Messwerten, Freigabe-Ampel grün/gelb/rot, automatische Vollständigkeitsprüfung
🖨
Arztübergabe
14-Tage Verlaufsübersicht druckfertig, Medikamentenplan, Beatmungsverordnung, Notfallplan integriert
💊
Medikamentenplan
Nüchtern-Hinweise, Wechselwirkungen, BTM-Kennzeichnung mit Vier-Augen-Prinzip. Akustische Warnung 10 Minuten vor BTM-Gabe. Zwei Kürzel erforderlich zur Bestätigung.
🫁
Beatmungsverordnung
IPAP/EPAP/Vt, Alarmgrenzen, Sauerstofftherapie Soll/Bedarf, Trachealkanülen-Management mit Wechseldatum
📋
SIS
Strukturierte Informationssammlung nach gesetzlichem Standard, 6 Themenfelder mit Ampel-Status und Fortschrittsanzeige
📝
Pflegebericht
Chronologisches Journal, 6 Farbkategorien (Beatmung/Pflege/Besonderheit/…), Filter nach Kategorie und Datum
⚠
Ereignisdokumentation
Sturzprotokoll, Beatmungszwischenfälle, vollständige Ereignisdokumentation mit Sofortmaßnahmen und Folgemaßnahmen
🩹
Wunddokumentation
Tracheostomapflege, Dekubitus, Verlaufsdokumentation mit Ampel-Status, Foto-Platzhalter für Bilddokumentation
🏥
Therapien & Dokumente
Physio/Ergo/Logo Verlauf, Arztbriefe, Befunde, Überweisungen mit Vorschau und Druckfunktion
🔗
FHIR Vorschau
Patientendaten als FHIR R4 Ressourcen — Patient, Observation, MedicationRequest, CarePlan. Interoperabilität demonstriert.
📊
Qualitätsdashboard
Management-Sicht: Dokumentationsquote, Compliance-Kurven, Alarmereignisse, Übergabequalität nach Pflegekraft.
Technischer Aufbau
Bewusst schlanker Stack — maximales Verständnis, minimale Abhängigkeiten
Backend
🐍
Python / Flask
REST API, Session-Management, JSON-Datenspeicherung, Compliance-Engine als eigenständiges Modul, Jinja2-Templates
Frontend
🎨
HTML / CSS / JavaScript
Responsives Design ohne Framework, Chart.js für Vitalwert-Kurven (14-Tage-Verlauf), alle Modals und Interaktionen nativ
Deployment
☁
GitHub + Render.com
Versionskontrolle mit Git, automatisches Deployment bei Push, kostenlos gehostet, CI/CD-Pipeline
Standard
🔗
FHIR R4 — HL7 Interoperabilität
Internationaler Healthcare-Interoperabilitätsstandard. CareBridge demonstriert Patient, Observation, MedicationRequest und CarePlan als standardkonforme FHIR R4 Ressourcen.
💡 Produktivszenario: In einer Produktivumgebung würde CareBridge auf einer zertifizierten Healthcare-Cloud laufen (z.B. Azure Health Data Services), mit echter Datenbank (PostgreSQL), DSGVO-konformer Datenspeicherung und einem Login-System mit Rollenkonzept (Pflegekraft · Arzt · Administration). Die aktuelle Architektur ist bewusst so gestaltet, dass dieser Umbau ohne Neuschreiben der Geschäftslogik möglich wäre.

Ich habe dieses Projekt gebaut, weil ich die Lücke zwischen Pflegepraxis und Digitalisierung aus eigener Erfahrung kenne. In der außerklinischen Beatmungspflege entstehen Fehler nicht nur durch fehlende Systeme — sondern auch durch mangelnde Struktur, fehlende Kontrolle und lückenhafte Dokumentation im Alltag. CareBridge begegnet genau diesem Problem: automatische Warnungen, Zeitstempel bei jeder Aktion, Vier-Augen-Prinzip bei BTM-Gaben und eine Compliance-Engine, die Lücken sichtbar macht bevor sie zur Gefahr werden.

Dieses Projekt habe ich gemeinsam mit Claude und Claude Code von Anthropic entwickelt. Ich verstehe die Anwendung, kann sie erklären und gezielt weiterentwickeln — die technische Umsetzung erfolgt durch KI auf meine fachliche Anforderung hin. Für mich zeigt das wie moderne Werkzeuge es ermöglichen, Fachwissen aus der Pflege direkt in digitale Lösungen zu übersetzen — ohne dass die Fachkompetenz an der Technik scheitert.
